Virtual File System (VFS)

ownCloud bietet den Nutzern die Möglichkeit, bei der Synchronisierung von Daten ein virtuelles Dateisystem (VFS) zu aktivieren. Das VFS stellt dem Benutzer alle Dateien in seinem Speicherbereich lokal zur Verfügung - allerdings nur über Platzhalter. Erst wenn der Benutzer eine Datei öffnet wird diese im Hintergrund automatisch heruntergeladen. Dies hat eine enorme Platzersparnis zur Folge und macht eine manuelle selektive Dateisynchronisation obsolet.

Hinweis

VFS ist aktuell nur für den Windows Sync-Client verfügbar. Für Mac und Linux sind benötigte Schnittstellen leider bisher nicht implementiert was die Nutzung von VFS verhindert.

Das Icon zeigt Ihnen an, dass diese Datei als Platzhalter auf Ihrem lokalen Dateisystem existiert. Klicken Sie doppelt auf eine Datei um diese zu öffnen. Sie werden sehen, dass sie das Icon in geändert wird. Die Datei ist jetzt heruntergeladen und kann lokal bearbeitet werden. Änderungen werden automatisch in die Cloud synchronisiert.

Sie können Dateien oder auch ganze Verzeichnisse herunterladen und damit auch offline verfügbar machen im dem Sie auf eine Datei rechts klicken und "Immer auf diesem Gerät beibehalten" wählen. Möchten Sie diesen Prozess rückgängig machen um Speicherplatz auf Ihrem lokalen Gerärt freizugeben wählen Sie im Kontextmenü "Speicherplatz freigeben". Damit wird die lokale Kopie gelöscht und wieder per Platzhalter vorgehalten.

Hinweis

VFS ist für Windows standardmäßig aktiviert. Damit sind Verzeichnisse und Dateien nicht automatisch offline verfügbar. Wie Sie Dateien und Ordner offline verfügbar machen wird Ihnen in EG-CS-14 erklärt.
